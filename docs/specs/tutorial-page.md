# Tutorial ページ仕様書 (v2)

## 概要

初心者がはじめてのPull Requestを作成するためのステップバイステップガイド。
ブラウザのみで完結する Web UI 版。

## URL

`/tutorial`

## デザイン

- Tailwind CSSユーティリティクラスによるスタイリング
- カラーパレット: Primary `#2563EB` / Accent `#10B981` / BG `#F8FAFC` / Text `#1E293B` / Sub `#64748B`
- トップページと統一されたデザイン言語

## ページ構成

### 1. ヒーローエリア

- タイトル: 「はじめてのPRチュートリアル」
- サブコピー: 「絵文字を1つ追加するだけ！あなたの最初のPull Requestを作成しましょう。」
- グラデーション背景、`pt-28` でヘッダー分のスペース確保

### 2. 前提条件セクション

| 項目 | 説明 |
|------|------|
| GitHub アカウント | github.com で無料作成可能 |
| ブラウザ | これを読んでいるなら準備OK |

※ Gitのインストールは不要

### 3. ステップガイド（9ステップ）

各ステップは `StepGuide` コンポーネントで表示。番号バッジ（青丸）、白背景カード、接続線で構成。

| Step | タイトル | 内容 |
|------|---------|------|
| 1 | リポジトリをForkする | Forkの意味、リポジトリへのリンク |
| 2 | emojis.txt を開く | data/emojis.txt の場所 |
| 3 | 編集モードに入る | 鉛筆アイコンの説明 |
| 4 | 好きな絵文字を追加する | 絵文字の追加方法、サイズ説明（1-3個） |
| 5 | 変更をコミットする | Commit changes ボタン、ブランチ作成の選択 |
| 6 | 元のリポジトリからPull Requestを作成する | Compare & pull request |
| 7 | PRの内容を確認して送信する | Create pull request |
| 8 | CIチェックを待つ | チェック項目の説明 |
| 9 | マージを待つ | 完了メッセージ |

### 4. Step 4 絵文字サイズ説明

| 入力 | カードサイズ |
|------|------------|
| `🐱` | 小さいカード |
| `🐱🐱` | 中くらい |
| `🐱🐱🐱` | 大きいカード |

### 5. 完了セクション

- 「PRがマージされたら...」メッセージ
- トップページへのCTAボタン

### 6. FAQ（8問）

| # | Q | A |
|---|---|---|
| 1 | GitHubアカウントが必要ですか？ | はい、github.comで無料作成 |
| 2 | Forkって何ですか？ | リポジトリの個人コピー |
| 3 | CIチェックが失敗した場合は？ | エラーメッセージを確認して修正 |
| 4 | PRがマージされない場合は？ | レビューコメントを確認して修正 |
| 5 | コンフリクトが起きた場合は？ | Forkを最新に同期してから再編集 |
| 6 | CLIでやりたい場合は？ | チュートリアル下部を参照 |
| 7 | 「絵文字以外の文字」エラーが出た | 絵文字だけを入力、テキストや記号は不可 |
| 8 | 「他の人の絵文字が削除されています」エラーが出た | 最後に追加のみ、他行は編集しない |
