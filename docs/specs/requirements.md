# 要件定義書: Git Training Ground

## 1. プロジェクト概要

### 1.1 目的
Git/GitHub 未経験者が「はじめてのPull Request」を安全に体験できるOSSプロジェクト。

### 1.2 ターゲットユーザー
- Git/GitHub を使ったことがない初心者
- OSSコントリビューションに興味があるが、踏み出せない人
- 40代前後の社会人エンジニア（キャリアチェンジ層含む）

### 1.3 解決する課題
| 課題 | 解決策 |
|------|--------|
| PRの作り方がわからない | ステップバイステップのチュートリアル |
| 失敗が怖い | contributors.json のみを編集する安全な設計 |
| 達成感がない | マージ後にトップページにカードが表示される |
| バリデーションが難しい | CIで自動チェック、エラーメッセージで案内 |

## 3. 非機能要件

### 3.1 パフォーマンス

| ID | 要件 | 基準 |
|----|------|------|
| NF-001 | SSG（静的サイト生成） | ビルド時にHTMLを生成 |
| NF-002 | 初回ロード | Lighthouse Performance 90+ |

### 3.2 アクセシビリティ

| ID | 要件 | 基準 |
|----|------|------|
| NF-010 | セマンティックHTML | heading階層、ランドマーク |
| NF-011 | aria属性 | FAQ accordion の aria-expanded |
| NF-012 | キーボード操作 | ボタン要素はフォーカス可能 |

### 3.3 ブラウザ対応

| ブラウザ | バージョン |
|---------|-----------|
| Chrome | 66+ |
| Safari | 13.1+ |
| Firefox | 63+ |
| Edge | 79+ |

### 3.4 セキュリティ

| ID | 要件 |
|----|------|
| NF-020 | NGワードフィルタリング（全角半角正規化対応） |
| NF-021 | XSS防止（Reactのデフォルトエスケープ） |
| NF-022 | 外部リンクに `rel="noopener noreferrer"` |

## 4. ユーザーフロー

```
初回訪問
  → トップページ（Hero → Features → HowItWorks → EmojiGrid → FinalCTA）
  → 「チュートリアルを始める」CTA
  → チュートリアルページ
  → 前提条件確認
  → Step 1〜9 を順に実施
  → PR作成・CIパス・マージ
  → トップページに自分のカードが表示される
```

## 5. 制約事項

- ユーザーが編集するファイルは `data/contributors.json` のみ
- サーバーサイドの動的処理なし（SSG完結）
- 外部APIへの依存なし
- PRのマージは人間（メンテナー）が行う
