# 要件定義書: Git Training Ground (v2)

## 1. プロジェクト概要

### 1.1 目的
Git/GitHub 未経験者が「はじめてのPull Request」を安全に体験できるOSSプロジェクト。

### 1.2 ターゲットユーザー
- Git/GitHub を使ったことがない初心者
- OSSコントリビューションに興味があるが、踏み出せない人
- 40代前後の社会人エンジニア（キャリアチェンジ層含む）

### 1.3 解決する課題
| 課題 | 解決策 |
|------|--------|
| PRの作り方がわからない | ステップバイステップのチュートリアル |
| 失敗が怖い | emojis.txt のみを編集する安全な設計 |
| 達成感がない | マージ後にトップページにカードが表示される |
| バリデーションが難しい | CIで自動チェック、初心者向けエラーメッセージで案内 |

### 2.4 CI バリデーション詳細

#### validate-emojis.mjs のチェック項目

| チェック | 判定方法 | エラー種別 |
|---------|---------|-----------|
| 絵文字のみ | 各 grapheme cluster が `\p{Extended_Pictographic}` を含む | `not-emoji` |
| 文字数制限 | `Intl.Segmenter` で grapheme cluster 1-3 個 | `too-many` |
| 同一絵文字 | 2文字以上の場合は全 grapheme が同一 | `mixed-emoji` |
| 空行 | スキップ | — |

#### validate-pr.yml のステップ

| ステップ | 目的 | 失敗時のメッセージ |
|---------|------|--------------|
| ファイルスコープ | emojis.txt 以外の変更検出 | 「emojis.txt のみ変更してください」 |
| 絵文字バリデーション | `node scripts/validate-emojis.mjs` | スクリプトのエラーメッセージ |
| 削除保護 | 既存行の削除・変更検出 | 「他の人の絵文字が削除されています」 |
| 追加制限 | 1 PR = 最大 1 行追加 | 「1行だけ追加してください」 |
| ビルド確認 | `npm run build` | ビルドエラー |

## 4. ユーザーフロー

```
初回訪問
  → トップページ（Hero → Features → HowItWorks → EmojiGrid → FinalCTA）
  → 「チュートリアルを始める」CTA
  → チュートリアルページ
  → 前提条件確認
  → Step 1〜9 を順に実施
  → PR作成・CIパス・マージ
  → トップページに自分の絵文字カードが表示される
```

## 5. 制約事項

- ユーザーが編集するファイルは `data/emojis.txt` のみ
- サーバーサイドの動的処理なし（SSG完結）
- 外部APIへの依存なし
- PRのマージは人間（メンテナー）が行う
